## The Core Design System

### Slide Chrome (every slide)
- **Background:** pure white `#ffffff`
- **Header bar:** thick dark charcoal bar spanning full width at the top of the slide — `#2d2d2d` approx
- **Footer bar:** same dark charcoal bar spanning full width at the bottom
- **No eyebrows, no decorative rules, no icons** — the bars are the only chrome

### Typography
- **Main text:** very heavy / black-weight sans-serif — looks like Impact or a very bold condensed font
- **Size:** massive — single-statement slides use font large enough to fill 30–50% of slide height
- **Color:** pure black `#000000`
- **Alignment:** left-aligned, with left padding
- **Supporting text:** smaller, regular weight, same left alignment below the headline

### Color Accents
- **Green** (`#22c55e` approx) — used for "this is good / positive outcome" items in lists
- **Red** (`#ef4444` approx) — used for "this is bad / old way / problem" items
- **Yellow/green highlight** — occasionally used on specific words to draw attention
- No yellow brand color, no gradient, no decorative color blocks

## Slide Types — Full Catalogue

### 1. Statement Slide (most common — ~60% of deck)
One sentence. Full stop. Huge bold text. White background with header/footer bars.

Examples:
- "So if you want to be the next success story..."
- "And another 2 every month after you don't"
- "Meaning there is a reason for the upside and for the downside... meaning no risk to you."
- "So we are limiting this to 7 'fast' spots total"
- "Transparently... there is only so much time in each day. Im actively still running this model myself."
- "...where EVERYONE did... no social proof. No..."
- "...cl aims to get to 100k/Month"

**Design rule:** presenter talks, slide shows ONE thought. Next slide = next thought. Very fast pacing.

### 2. Title + Bullet List Slide
Bold title headline, then 5–8 bullet items. Items are color-coded:
- Green ✅ = good/positive
- Red ❌ = bad/problem

Example: "Who This Works Best For" — list of criteria with green checkmarks
Example: "Now I know this sounds Juicy... lets make sure its right for you" — split green/red columns

### 3. Comparison Table Slide
Two columns. Left = old way (red X items). Right = new way (green ✅ items).
Bold title: "What If There Was A Better Way"
Row-by-row comparison — same item, different outcome in each column.

### 4. Maths / Numbers Slide
Title: "Funnel Math For Creators"
Numbered rows with green-highlighted text showing revenue calculations.
Simple layout — title + rows of numbers, no charts or graphs.

### 5. Screenshot / Proof Slide
Title + short description text on the left.
Actual platform screenshot on the right (calendar, CRM, dashboard, WhatsApp).
Example: "For Example: We Only Work With 6-10 Full Stack Clients At A Time" — shows a calendar/scheduling tool screenshot

### 6. Testimonial Slide
Title: "As Well As For [Name]"
Short paragraph of result text (green-highlighted key result).
Video thumbnail or photo of the client embedded on the right side.

### 7. Path / Choice Slide
"2 Paths Moving Forward" — two side-by-side boxes.
Each box: label + bullet list of what that path looks like.
Visual split, not a table.

### 8. Video Embed Slide
YouTube-style thumbnail embedded large in the slide.
Title or description text alongside.
Used mid-deck for social proof or demonstration.

### 9. Navigation / Tool Screenshot
Shows actual software being used (HighLevel, GHL, scheduling tools).
Title + description + screenshot side by side.
"...and decodes another right below" style — reveals the system being used.

### 10. Journey / Story Slide
Photo of a person + short transformation story text.
Green-highlighted result number.
Used for testimonials mid-deck and in proof section.

## Pacing Philosophy

- **~90 slides, ~60 minutes** = ~1 slide per 40 seconds
- One thought per slide — presenter advances as they speak
- NO slide holds multiple ideas
- Visual variety comes from switching between statement, list, screenshot, and comparison types — not from color or decoration
- The slide reinforces what the presenter just said — it doesn't replace them

---

## What Makes It Feel Different

| Scott Kelly | Standard webinar deck |
|---|---|
| One statement per slide | Multiple bullets per slide |
| Massive bold text | Normal heading + body text hierarchy |
| Real screenshots as proof | Generic stat cards |
| Green/red color coding | Single accent color |
| White + dark bars = all the chrome | Eyebrows, rules, icons, section headers |
| 90 slides for 60 mins | 30–40 slides for 60 mins |
| Slides follow the speech | Slides lead the speech |

## How to Apply to Dan's Deck

1. **More statement slides** — many of Dan's bullets should each become their own slide
2. **Dark header + footer bars** as the slide chrome instead of eyebrow labels
3. **Green/red comparison tables** for Old Way vs AIO Way
4. **Real screenshot embeds** — Dan's actual dashboard, HighLevel, creator revenue reports
5. **Maths slide** using the numbered row format with green highlighting
6. **Testimonials** with video thumbnail + result text layout
7. **One idea per slide** — if a bullet list has 4 items, consider making 4 slides
8. **Font weight** — move from Inter 900 to something heavier and more condensed
